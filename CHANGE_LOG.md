* 2026.07.22 (v1.3.1): make the 3D encoder correctly invariant to rigid motions — the legacy first layer built its Gram matrix from absolute atom positions (`MolConv1`), which is only O(3)-invariant and breaks under translation; `MolConv2` instead builds it from relative displacements `<(xj−xi), (xk−xi)>` (which encode the angle ∠jik), centers coordinates on the real-atom centroid, and masks zero-padding atoms out of real atoms' kNN, giving full **E(3)** invariance (rotation + reflection + translation), verified in `tests/test_se3_invariance.py` and `tests/test_encoder_invariance.py`; the bare name `MolConv` is now a soft link to `MolConv2` (the default) and the legacy encoder is available explicitly as `MolConv1`; an optional `chirality` channel — a signed-volume pseudoscalar — makes the first layer reflection-*sensitive* (**SE(3)**) for chiral tasks such as 3DMolCSP; select with `encoder_version` (1|2) and `chirality` in the model config; document the encoder and its invariance in `docs/source/encoder.md`; add optional self-supervised 3D pretraining of the encoder via masked distance reconstruction — mask 15% of atoms (zeroing all 21 input dimensions including xyz) and predict Euclidean distances between (masked, unmasked) atom pairs from per-atom encoder features, forcing the encoder to infer 3D geometry from chemical context (`MolNet_SSL`, `MolSSL_Dataset`, and `Encoder.return_per_atom`, which exposes per-atom features to the pretraining head); cap ETKDGv3 `maxIterations` to prevent hangs on difficult ring systems and add `MMFF` as a conformation option; fix spectrum normalization to be per-spectrum rather than per-batch; fix a missing mask key in augmented data entries; move training hyperparameters into configuration files and remove the redundant `batch_size` from model config sections; support batched MS/MS inference (`batch_size`) and in-memory input (`load_dataframe`, `load_smiles`); fix `pred_msms` to reload weights when the instrument or checkpoint changes; cache downloaded checkpoints in a per-user directory (`platformdirs.user_cache_dir("molnetpack")`, e.g. `~/.cache/molnetpack`) with a `$MOLNETPACK_HOME` override and a backward-compatible fallback to an in-package `check_point/`, harden the download (streaming, timeout, status check, cleanup), and delete the `.zip` after extraction; `plot_msms` writes `.png` files and draws 2-D depictions directly.
